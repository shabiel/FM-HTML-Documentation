Global File Structure

KEY File 

Version 22 of VA FileMan introduces a way to uniquely identify a record on a file. The developer defines a field or fields as belonging to a KEY. The developer must also build an index for those fields. Fields in the Primary KEY are displayed during classic FileMan lookup ^DIC. KEY fields are used to decide whether a record already exists on the target file during Transfer or during data Installation using the Kernel Installation and Distribution System (KIDS). The KEY file is #.31 (stored in global ^DD("KEY",). See also INDEX File.
